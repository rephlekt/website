What is Continuous Integration?

  • The use of an automated build server that listens for code changes in a repository
  • Build server will build/compile the code, and run automated tests on the output

What is Continuous Delivery?

  • Built software can be released whenever needed
  • Releases are multiple time a day, not a few months between like the old days
  • Automated deployment server will send product over to multiple app servers

Difference between Continuous Delivery and Continuous Deployment:

  • Continuous Delivery requires a manual step to approve a deployment
  • Continuous Deployment is fully automated, no manual intervention of approvals
